Operational Protocols - System Reference
01 AI Deterrence Sequence
When a sensor triggers while the system is armed, VOXWALL enters a 10-second entry delay. During this window, a countdown is displayed on the dashboard. If the system is not disarmed within that window, the AI voice agent activates and issues a verbal warning to the intruder. The agent will engage up to 3 times. If the threat persists through all three dialogues, the 12V siren relay activates. At any point, pressing TERMINATE ALERT or disarming from the dashboard ends the sequence immediately.
02 Entry Delay Window
The entry delay is a 10-second grace period that activates when an alarm-triggering sensor fires while the system is armed. It is designed for authorized users re-entering the premises - navigate to the dashboard and press DISARM before the countdown reaches zero. If you miss the window, the AI deterrence phase will begin. You can still disarm the system during the AI phase or after the siren engages using the dashboard.
03 Siren Engagement
The 12V siren activates only after the AI deterrence phase is exhausted - three unsuccessful verbal engagements with no disarm response. Once active, the siren drives the relay continuously until disarmed from the dashboard. The relay is wired active-LOW: the signal is held HIGH at rest to ensure the siren remains off even if the ESP32-S3 restarts unexpectedly. Siren events are logged automatically in the Event Log under SIREN_ACTIVE.
04 Remote Disarm Procedure
Open the dashboard on any device and press DISARM. The command is sent to the backend over HTTPS, forwarded to the ESP32-S3 on the next heartbeat poll (within ~2 seconds), and the system transitions to DISARMED. The relay is de-energised, the AI agent loop exits, and any active siren stops. The status banner on your dashboard updates in real-time via WebSocket. No app installation required - the dashboard runs in any modern browser.
05 Sensor Node Registration
Sensor nodes (ESP32-C3) register automatically the first time they transmit over ESP-NOW to the central unit (ESP32-S3). The central unit forwards the node identity to the backend, which creates a sensor record with status ONLINE. Nodes appear in the Sensors page immediately after first contact. If a node goes offline (no heartbeat for 60 seconds), its status transitions to OFFLINE automatically. Re-powering the node triggers re-registration with no manual action required.
06 Arm Modes Explained
ARM AWAY - Full perimeter active. All sensor types (motion, door, vibration) will trigger the alarm sequence. Use when the premises are unoccupied.

ARM HOME - Interior-safe mode. Perimeter sensors remain active while interior motion sensors are suppressed. Use when occupants are present inside.

ARM NIGHT - Night-profile perimeter mode. Similar to ARM HOME but optimised for nighttime sensitivity thresholds. Designed for sleeping hours when occupants are stationary.
